RELEASE NOTES - v7.0.x Series
==============================

v7.0.3 (March 11, 2026) - SAFETY ENHANCED
------------------------------------------
🔒 Communication Watchdog
   - 60-second timeout monitoring
   - Auto-abort test if no serial data received
   - Prevents runaway discharge on PC disconnect/crash
   - Displays "Comm Timeout - PC Disconnected" message

💓 Heartbeat Protocol
   - Python app sends HB command every 30 seconds
   - Transparent to user (silently processed)
   - Keeps watchdog alive during long tests
   - Watchdog updated on: AUTO_BT, params, cancel, HB

🛡️ Enhanced Safety
   - Critical protection for unattended operation
   - Automatic load shutdown on communication loss
   - Battery protection from over-discharge

v7.0.2 (March 10, 2026) - UX ENHANCED
--------------------------------------
🎯 Auto Mode Switch
   - Listens for AUTO_BT command on serial port
   - Automatically switches to Battery Test mode
   - Sends ACK_BT acknowledgment when ready
   - Seamless integration with Python app v2.0.h+

🔄 Backward Compatibility
   - Works with old Python app (manual mode selection)
   - Works with new Python app (auto mode switch)
   - No breaking changes to protocol

v7.0.1 (March 10, 2026) - FEATURE COMPLETE
-------------------------------------------
⏱️ Recovery Monitoring
   - Configurable timeout: 1-30 minutes
   - Battery voltage monitoring during recovery
   - Real-time countdown display
   - Parameter 9 in serial protocol

📊 Filtered Cutoff Detection
   - 16-sample moving average voltage filtering
   - Prevents premature cutoff from noise
   - Robust end-of-discharge detection
   - Eliminates false positives

🏗️ PlatformIO Migration
   - Complete C++ conversion from Arduino IDE
   - Modular code organization
   - All function bodies implemented
   - Comprehensive testing completed

v7.0.0 (March 8, 2026) - BASE RELEASE
--------------------------------------
⚡ Dual-core optimization - ISR core affinity fixes
⚡ Enhanced task priority hierarchy (encoder: Priority 11)
⚡ Increased OLED display stack size (4KB → 8KB)
⚡ Comprehensive thread safety with mutex protection
⚡ Fixed BatteryMode termination message logic

Performance Improvements:
------------------------
- ISRs now attach on Core 0 for efficient signaling (eliminates cross-core overhead)
- Encoder response is now immediate with higher priority
- Graphics rendering more stable with larger stack buffer

Thread Safety:
-------------
- All shared variables protected with portMUX_TYPE mutex
- Fixed race conditions in ADC reading (dutV, shuntV)
- Protected display variable copying (dispVoltage, dispCurrent)
- Safe temperature and power reads across all modules

Bug Fixes:
----------
- Battery Mode: Prevented duplicate termination messages
- Rotary Encoder: Fixed critical section types (ISR vs task)
- OLED Display: Fixed race conditions in setup and temperature display
- Main Loop: Complete protection for checkValidValues()

Hardware Support:
----------------
- ESP32 DevKit (dual-core Xtensa LX6 @ 240MHz)
- ADS1115 16-bit ADC (dynamic gain: 0.5-105V range)
- DAC8571 16-bit DAC (0-10.5A output)
- SSD1351 128x128 OLED Display
- Rotary Encoder with ISR processing
- PWM Fan Control (temperature-based)

Resource Usage:
--------------
RAM:   7.2% (23,480 / 327,680 bytes)
Flash: 25.8% (337,877 / 1,310,720 bytes)

Operating Modes:
---------------
- CC: Constant Current
- CV: Constant Voltage
- CP: Constant Power
- CR: Constant Resistance
- Battery: Discharge test with PC app communication

Communication:
-------------
- Serial: 9600 baud
- Protocol: MSGST...MSGEND format
- Compatible with Battery Tester Python app

Version History:

Compatibility:
-------------
✅ Python Battery Tester v2.0.h or later (recommended for all features)
✅ Python Battery Tester v2.0.0 - v2.0.g (works, but manual mode selection)
✅ Backward compatible with older Python versions
⚠️ Python app v2.0.h+ required for heartbeat (otherwise 60s timeout)

Known Issues / Limitations:
--------------------------
- Recovery timeout maximum: 30 minutes (by design)
- Heartbeat dependency: Python app must send HB every 30 seconds
- No local data storage: all data handled by Python app
- Single channel: one battery at a time
