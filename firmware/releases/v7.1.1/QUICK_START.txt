ENTERING CALIBRATION MODE
──────────────────────────
1. Power off ESP32
2. Press and HOLD encoder button
3. Power on ESP32 (keep holding)
4. Wait for "CALIBRATION MODE" screen
5. Release button after screen appears

TIP: Start holding button when screen blanks from previous session


SERIAL SETUP
────────────
Baud Rate: 9600
Port: Check Device Manager (Windows) or ls /dev/ttyUSB* (Linux)
Terminal: PuTTY, Arduino IDE Serial Monitor, or any terminal


CALIBRATION COMMANDS
────────────────────
Type "CAL" alone to show full command menu

CAL SHOW              - Display current calibration values
CAL CV <value>        - Set CV mode calibration factor
CAL DUTV <value>      - Set DUT voltage calibration factor  
CAL DUTC <value>      - Set DAC-ADC calibration point (mV)
CAL SHUNT <value>     - Set current display calibration factor
CAL SAVE              - Save changes to ESP32 memory (REQUIRED!)
CAL RESET             - Restore factory defaults
CAL EXIT              - Exit calibration (requires power cycle)

Commands are case-insensitive (CAL or cal both work)
Backspace and Ctrl-C supported for editing


CALIBRATION PARAMETERS
──────────────────────
• cvCalFactor (default 1.0)
  Most commonly adjusted - affects CV mode trigger point
  Range: typically 1.0 to 1.10
  Calibrate at 50V for best accuracy

• dutVcalib (default 1.0)
  Adjusts DUT voltage display accuracy
  Range: typically 0.99 to 1.01

• shuntVcalib (default 2.5000)
  Adjusts current display accuracy
  Range: typically 2.45 to 2.55

• DUTCurrent (default 400.00)
  Factory DAC-ADC calibration point in mV
  Usually doesn't need adjustment


CALIBRATION EXAMPLE - CV MODE
──────────────────────────────
Problem: CV mode triggers at wrong voltage

1. Enter calibration mode (hold button during boot)
2. Connect serial terminal at 9600 baud
3. Type: CAL SHOW
4. Note current cvCalFactor (e.g., 1.0)
5. Set test voltage: 50.0V @ 100mA constant current
6. Measure actual trigger voltage: e.g., 53.03V
7. Calculate: cvCalFactor = 53.03 / 50.0 = 1.0606
8. Type: CAL CV 1.0606
9. Type: CAL SAVE
10. Type: CAL EXIT
11. Power cycle ESP32
12. Test: Set 50V, verify triggers at ~50.0V (±0.04V)


VOLTAGE DISPLAY CALIBRATION
────────────────────────────
Problem: Voltage display reads high by 0.5%

1. Apply known voltage: 10.00V from calibrated source
2. DL displays: 10.05V
3. Calculate: dutVcalib = 10.00 / 10.05 = 0.9950
4. Enter calibration mode
5. Type: CAL DUTV 0.9950
6. Type: CAL SAVE
7. Type: CAL EXIT
8. Power cycle and verify: 10.00V shows correctly


CURRENT DISPLAY CALIBRATION
────────────────────────────
Problem: Current display reads low

1. Set 1.000A with calibrated ammeter in series
2. DL displays: 0.980A
3. Calculate: shuntVcalib = current * (displayed / actual)
4. If current shuntVcalib = 2.5000:
   New value = 2.5000 * (1.000 / 0.980) = 2.5510
5. Enter calibration mode
6. Type: CAL SHUNT 2.5510
7. Type: CAL SAVE
8. Type: CAL EXIT
9. Power cycle and verify: 1.000A shows correctly


EXITING CALIBRATION MODE
─────────────────────────
Two methods:

1. BUTTON CLICK EXIT (quick cancel)
   - Click encoder button during calibration
   - No power cycle needed
   - Continues to normal boot immediately
   - Use when: Accidental entry, no changes made

2. CAL EXIT COMMAND (formal completion)
   - Type: CAL EXIT in serial terminal
   - Requires power cycle to restart
   - Use when: Calibration work completed


IMPORTANT NOTES
───────────────
✓ Case-insensitive: "CAL", "cal", "Cal" all work
✓ MUST use CAL SAVE to persist changes!
✓ Calibration survives firmware updates
✓ Fan automatically stops during calibration mode
✓ Backspace key works for editing commands
✓ Ctrl-C cancels current line input
✓ Echoes characters as you type
✓ Shows ">" prompt after Enter

✗ Changes NOT saved until you type CAL SAVE
✗ Pressing button cancels without saving
✗ Power cycle during calibration loses unsaved changes


TROUBLESHOOTING
───────────────
Can't enter calibration mode:
  → Start holding button as screen blanks from previous session
  → Hold through entire boot sequence (2-3 seconds)
  → Button must be pressed BEFORE splash screen appears

Fan still running:
  → Fixed in v7.1.1! Fan should stop immediately
  → If not, verify you're running v7.1.1 firmware

Commands not recognized:
  → Check baud rate is 9600
  → Type "CAL" alone to show menu
  → Commands must start with "CAL" or "cal"

Values not saving:
  → Must type "CAL SAVE" after changing values
  → Serial should confirm: "Calibration saved to Preferences"
  → Use "CAL SHOW" to verify values persisted

Serial not connecting:
  → Close other programs using serial port
  → Check correct COM port selected
  → Install CP210x or CH340 drivers if needed
  → Try lower baud rate if garbled characters


VERIFICATION AFTER CALIBRATION
───────────────────────────────
1. Power cycle ESP32
2. Enter calibration mode again
3. Type: CAL SHOW
4. Verify values match what you set
5. If correct, calibration is persistent ✓
